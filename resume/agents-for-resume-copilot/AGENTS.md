# 约定
- 项目语言为中文，如无特殊要求，对话、注释、文档使用中文书写

# 目录结构
/backend 后端代码  
/web 前端页面代码  
/docs 跨端业务逻辑与数据库文档

# 任务执行前要做
1. 先读本文件（`AGENTS.md`）
2. 阅读要修改的项目目录下的 `AGENTS.md`

# 任务执行后要做
- 新增/修改了业务逻辑，必须同步更新 `docs/business_logic/` 对应文档
- 修改数据库定义，必须同步更新 `docs/database/` 对应 SQL

# 工具执行注意事项

- 枚举/搜索文件时默认排除 `.venv`、`__pycache__`、`.pytest_cache` 等依赖和缓存目录，避免无关输出过大。
- 执行会连接本地数据库、外部数据库、模型服务、HTTP 服务或其他网络/本机 socket 的命令时，第一次执行就需要申请提权执行，不要先在沙箱中试跑或等失败/超时后再提权；包括但不限于 `backend/scripts/*_smoke.py`、FastAPI/TestClient 冒烟脚本、`psql` 迁移或验证命令

# 代码协作约定
- 避免提交“无意义代码”
- 以下类型的新增代码需要添加注释：
  - 枚举：含义
  - 常量：含义
  - 全局变量：含义、效果
  - 方法：功能、应该被使用的场景说明
- commit 信息遵守 Conventional Commits 要求
- 不要进行代码格式化与 imports 优化，代码提交时会自动执行
